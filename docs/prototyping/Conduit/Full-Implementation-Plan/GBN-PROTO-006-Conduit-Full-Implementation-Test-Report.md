## 7. Known Open Items

- Full live mobile-carrier validation has not been run in this report yet.
- The AWS validation captured here was intentionally minimal: `DesiredBridgeCount=1`, no scale run, no controlled multi-bridge churn, and no real mobile carrier path.
- The smoke deployment used `GBN_BRIDGE_POSTGRES_TLS_ACCEPT_INVALID_CERTS=true` as a development escape hatch for RDS TLS certificate trust inside the slim container images. Production validation should supply an RDS CA bundle with `GBN_BRIDGE_POSTGRES_TLS_CA_PEM` or `GBN_BRIDGE_POSTGRES_TLS_CA_FILE` and keep invalid certificate acceptance disabled.
- A chain-specific AWS trace run with `collect-conduit-traces.sh --chain-id <id> --require-chain-id` is still needed for final mobile/live evidence.
- Mobile carrier / network observations must be added by the operator running the live test.
